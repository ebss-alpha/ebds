Accessibility statement

This accessibility statement applies to content published on the "{{ serviceName }}" service.

This website is run by the Department for Energy Security and Net Zero.

We want as many people as possible to be able to use this website. For example, this means you should be able to:

  • change colours, contrast levels and fonts
  • zoom in up to 300% without the text spilling off the screen
  • navigate most of the website using just a keyboard
  • navigate most of the website using speech recognition software
  • use most of the website using a screen reader (including the most recent versions of JAWS, NVDA and VoiceOver)

We've also made the website text as simple as possible to understand

AbilityNet has advice on making your device easier to use if you have a disability.

How accessible this website is

We know some parts of this website are not fully accessible:

  • form inputs lack accessible titles
  • a correctly formed heading structure is not present to facilitate navigation
  • some page content is not identified with appropriate ARIA landmarks
  • some buttons and links do not have an accessible name
  • page titles are not descriptive
  • a visible focus indicator is not always present when tabbing page content
  • a text description is not provided for error fields on forms
  • suggestions for input errors are not provided
  • HTML validation errors are present that may impact navigating with assistive technologies
  • additional content which appears on screen is not announced to screen reader users
  • content is announced multiple times to users of screen readers
  • users of Dragon Naturally Speaking may not be able to upload documents

Feedback and contact information

If you need information on this website in a different format like accessible PDF, large print, easy read, audio recording or braille, contact us by:

  • email: support@ebds.beis.gov.uk
  • phone: 0300 400 5251

As part of providing this service, we may need to send you messages or documents. We'll ask you how you want us to send messages or documents to you, but contact us if you need them in a different format. For example, large print, audio recording or braille.

Reporting accessibility problems with this website

We're always looking to improve the accessibility of this website. If you experience a problem not listed on this page or think we're not meeting accessibility requirements, contact us: gov.uk/contact-beis

Enforcement procedure

The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018 (the 'accessibility regulations'). If you're not happy with how we respond to your complaint, contact the Equality Advisory and Support Service (EASS).

Contacting us by phone or visiting us in person

We provide a text relay service for people who are D/deaf, hearing impaired or have a speech impediment.

Our offices have audio induction loops, or if you contact us before your visit, we can arrange a British Sign Language (BSL) interpreter.

Find out how to contact us at: www.gov.uk/guidance/contact-beis

Technical information about this website's accessibility

The Department for Energy Security and Industrial Strategy is committed to making its website accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

Compliance status

The service is not compliant with the Web Content Accessibility Guidelines version 2.1 AA standard. The non-compliances are listed below.

Non-accessible content

The content listed below is non-accessible for the following reasons.

Non-accessible content

Some form input fields lack an associated text label or descriptive title. Incorrectly associated labels do not provide functionality or information about the form control to screen reader users. This fails WCAG 2.1 success criterion 1.1.1 (Non-text Content).

We plan to resolve this by 7 June 2023.

Some regions of the page are not identified with appropriate landmarks. This may be confusing to assistive technology users who navigate using landmarks. This fails WCAG 2.1 success criterion 1.3.1 (Info and Relationships).

We plan to resolve this by 7 June 2023.

A correctly formed heading structure is not present to facilitate navigation across the site, with skipped and empty heading levels present. This fails WCAG 2.1 success criterion 2.4.1 (Bypass Blocks).

We plan to resolve this by 7 June 2023.

Some pages lack a descriptive title. This will prevent users from quickly and easily identifying whether the information contained within the page is relevant to their needs. This fails WCAG 2.1 success criterion 2.4.2 (Page Titled).

We plan to resolve this by 7 June 2023.

Some buttons on the page contain the same text with no additional information to help distinguish between the buttons for screen reader users. This fails WCAG 2.1 success criterion 2.4.4 (Link Purpose (In Context)).

We plan to resolve this by 7 June 2023.

A visible focus indicator is not always present when tabbing page content. This includes when the pop-up appears when providing evidence. This fails WCAG 2.1 success criterion 2.4.7 (Focus Visible).

We plan to resolve this by 7 June 2023.

Some mandatory fields within forms do not show an error message to identify that they require filling. This fails WCAG 2.1 success criterion 3.3.1 (Error Identification)

We plan to resolve this by 7 June 2023.

Some error messages when submitting forms does not identify how to correct the error within the form. This fails WCAG 2.1 success criterion 3.3.3 (Error Suggestion)

We plan to resolve this by 7 June 2023.

When evaluating some webpages, it's been identified that multiple HTML validation errors are present. Issues identified included, elements not being nested according to their specifications, as well as missing start and end tags. This fails WCAG 2.1 Success criterion 4.1.1 Parsing.

We plan to resolve this by 7 June 2023.

Some elements are inaccessible to users of Dragon Naturally Speaking. This fails WCAG 2.1 success criterion 4.1.2 (Name, Role, Value)

We plan to resolve this by 7 June 2023.

Whilst navigating the website using a screen reader, some content is announced multiple times. This may become confusing for some users. This fails WCAG 2.1 success criterion 4.1.2 (Name, Role, Value)

We plan to resolve this by 7 June 2023.

Some form validation messages are not announced by screen readers. This may result in some error messages being missed. This fails WCAG 2.1 success criterion 4.1.3 (Status Messages)

We plan to resolve this by 7 June 2023.

What we are doing to improve accessibility

We will carry out on-going internal audits to check the accessibility of this service as well as taking feedback from users.

Any required changes will be planned into our continous improvement work for this service and added to a backlog of improvement goals and prioritised according to user need and cost.

This accessibility statement will be updated based on any issues we identify or any changes we make to address any issues raised.
